Android Architecture - Overview

What is Android?

Android is an operating system that runs on billions of smartphones, tablets, and other devices worldwide. Think of it as the foundation that makes your phone work.

The Layer Approach

Android is built in layers, like a cake. Each layer has a specific job and works with the layers above and below it.

3

Android Architecture - Foundation Layers

Layer 1: Linux Kernel (The Foundation)

  • The bottom layer that talks directly to your phone's hardware
  • Manages battery, memory, screen, camera, and other physical components
  • Handles security and keeps different apps separated

Layer 2: Hardware Abstraction Layer (HAL)

  • Acts as a translator between Android and different phone manufacturers
  • Allows Android to work on Samsung, Google Pixel, and other brands
  • Makes it easier for developers to write apps that work on any Android device
4

Android Architecture - Upper Layers

Layer 3: Android Runtime and Libraries

  • Contains pre-written code that apps can use
  • Includes tools for graphics, databases, web browsing, and more
  • The Android Runtime runs your apps and manages memory efficiently

Layer 4: Application Framework

  • Provides ready-made components for app developers
  • Includes things like notification systems, location services, and content sharing
  • Saves developers from reinventing the wheel for common features

Layer 5: Applications

The apps you actually see and use: Gmail, Chrome, Camera, etc.

iOS Architecture - Overview

What is iOS?

iOS is Apple's operating system that runs exclusively on iPhones and iPads. It's known for smooth performance and tight integration with Apple hardware.

The Layered Design

Like Android, iOS uses a layered architecture. However, iOS is designed specifically for Apple devices, allowing deeper optimization.

6

iOS Architecture - Foundation Layers

Layer 1: Core OS Layer (The Foundation)

  • Directly manages iPhone hardware like processor, memory, and storage
  • Handles security, file systems, and low-level networking
  • Similar to Android's Linux kernel but optimized for Apple chips

Layer 2: Core Services Layer

  • Provides fundamental services like iCloud, location services, and databases
  • Manages networking, file handling, and data storage
  • Includes frameworks for security and privacy features
7

iOS Architecture - Upper Layers

Layer 3: Media Layer

  • Handles all graphics, audio, and video
  • Powers animations, image processing, and gaming graphics
  • Includes technologies like Metal (for high-performance graphics) and Core Audio

Layer 4: Cocoa Touch Layer

  • The top layer that app developers work with most
  • Provides user interface components like buttons, sliders, and gestures
  • Includes features like multitasking, notifications, and app extensions
  • Makes apps feel smooth and responsive
8

Android vs iOS Architecture

Key Differences

Android

  • Open source and runs on many device types
  • More flexibility for customization
  • Works with hardware from hundreds of manufacturers

iOS

  • Closed system designed for Apple devices only
  • Tighter hardware-software integration
  • Consistent experience across all Apple devices

Both approaches have advantages and create reliable, powerful mobile experiences.

Edge Computing - What Is It?

The Traditional Way: Cloud Computing

Traditionally, when your phone needs to process data, it sends it to distant data centers (the cloud), which process it and send results back. This can take time.

The New Way: Edge Computing

Edge computing brings the processing power closer to where the data is created. Instead of traveling thousands of miles to a data center, your data is processed nearby.

Think of it like: having a small convenience store in your neighborhood versus driving to a big supermarket in another town.

10

Edge Computing - How It Works

Where Is "The Edge"?

The edge refers to devices and servers located physically closer to users:

  • Cell phone towers with processing capabilities
  • Local servers in stores or buildings
  • Smart devices themselves (like security cameras with AI built-in)
  • Regional mini data centers

The Process

  1. Your device creates data (like a video from a security camera)
  2. Instead of sending it far away, it's processed at a nearby edge server
  3. Only important results are sent to the main cloud
  4. You get faster responses and use less bandwidth
11

Edge Computing - Benefits

Why Use Edge Computing?

  • Speed: Processing happens nearby, so responses are lightning-fast (important for self-driving cars or surgery robots)
  • Bandwidth Savings: Less data traveling across the internet means lower costs and less network congestion
  • Reliability: Works even if connection to the main cloud is slow or interrupted
  • Privacy: Sensitive data can be processed locally without leaving your location

Real-World Examples

Smart traffic lights, retail store analytics, industrial sensors, augmented reality apps

Serverless Platforms - Overview

What Does "Serverless" Mean?

A Confusing Name: Despite the name, serverless platforms DO use servers. The term means developers don't have to worry about managing those servers.

The Traditional Way

Before serverless, developers had to:

  • Buy or rent servers
  • Install and update software
  • Monitor server health 24/7
  • Scale up when traffic increases
  • Pay for servers even when not being used

The Serverless Way

Developers just write code. The platform automatically handles everything else.

13

Serverless Platforms - How They Work

The Function Approach

Instead of running a whole application on a server all the time, you break your application into small functions that run only when needed.

The Process

  1. You write a function (like "process a payment" or "resize an image")
  2. You upload it to a serverless platform like AWS Lambda, Google Cloud Functions, or Azure Functions
  3. The function sits idle, costing you nothing
  4. When someone triggers it (like uploading a photo), the platform instantly runs your function
  5. After finishing, it shuts down again
  6. You only pay for the exact time it ran
14

Serverless Platforms - Benefits

Key Advantages

  • Pay Only For What You Use: If your function runs for 200 milliseconds, you pay for 200 milliseconds
  • Automatic Scaling: If 1 person or 1 million people use your app, the platform handles it automatically
  • No Server Management: No updates, no security patches, no maintenance headaches
  • Faster Development: Focus on writing code, not managing infrastructure

Great For

Image processing, data transformation, chatbots, scheduled tasks, APIs, and event-driven applications

IoT Platforms - What Is IoT?

IoT = Internet of Things

IoT refers to everyday objects connected to the internet that can send and receive data.

Examples All Around Us

  • Smart home devices (thermostats, light bulbs, door locks)
  • Fitness trackers and smartwatches
  • Connected cars
  • Industrial sensors in factories
  • Smart agriculture sensors in farms
  • Medical monitoring devices

These devices create massive amounts of data that need to be collected, processed, and acted upon.

16

IoT Platforms - What They Do

The Challenge

Imagine managing thousands or millions of connected devices, each sending data constantly. How do you organize this chaos?

What an IoT Platform Provides

  • Device Management: Register, monitor, and update all your connected devices from one place
  • Data Collection: Gather data from all devices reliably, even with poor internet connections
  • Data Processing: Analyze the incoming data to find useful insights
  • Security: Protect devices from hackers and ensure data privacy
  • Integration: Connect IoT data to other systems and applications
17

IoT Platforms - Key Components

  • Connectivity Layer: Handles communication between devices and the platform (WiFi, Bluetooth, cellular, etc.)
  • Device Management: Monitors device health, sends updates, and manages device lifecycles
  • Data Processing Engine: Processes and analyzes incoming data streams in real-time
  • Application Layer: Where you build apps and dashboards to visualize and use the data
  • Security Layer: Encrypts data, authenticates devices, and prevents unauthorized access

Popular Platforms

AWS IoT, Google Cloud IoT, Microsoft Azure IoT, Arduino Cloud

18

IoT Platforms - Real-World Examples

Smart Home

Your thermostat learns your schedule and adjusts temperature automatically. The IoT platform collects data from temperature sensors and sends commands to your HVAC system.

Smart Factory

Sensors on machines predict when maintenance is needed. The platform analyzes vibration and temperature data to prevent breakdowns.

Smart Agriculture

Soil moisture sensors tell farmers exactly when to water crops. The platform processes data from hundreds of sensors across large farms.

Healthcare

Wearable devices monitor patient vital signs. The platform alerts doctors if readings become concerning.

How These Technologies Work Together

A Real-World Scenario: Smart City Traffic System

  • IoT Platform: Collects data from traffic cameras and sensors at intersections
  • Edge Computing: Processes video feeds locally at each intersection to count cars and detect accidents
  • Serverless Functions: Trigger alerts to emergency services when accidents are detected
  • Mobile Apps (Android/iOS): Display real-time traffic conditions to drivers and suggest alternate routes

All these technologies complement each other to create intelligent, responsive systems.

20

Another Example - Food Delivery App

  • Android/iOS Architecture: Provides the mobile app interface for customers and delivery drivers
  • IoT Platform: Tracks delivery vehicle locations and monitors food temperature in smart containers
  • Edge Computing: Processes location data quickly to provide real-time driver positions
  • Serverless Platform: Handles payment processing, order matching, and notification sending without maintaining expensive servers running 24/7

This combination creates a seamless experience from order to delivery.
